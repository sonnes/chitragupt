1. Install Command Definition & Flags (~/work/chitragupt/cmd/cg/install.go)

The CLI command accepts:

  • --agent / -a (default: "claude"): Agent name - currently only "claude" is supported
  • --format / --fmt (default: ["html"]): Output formats (repeatable) - html, json, markdown
  • --branch (default: "transcripts"): Name for the orphan branch

The command runs install.Run(cfg) and outputs a summary of what was installed.

2. End-to-End Install Flow (~/work/chitragupt/install/install.go)

The Run() function executes 5 sequential steps:

Step 1: Create Orphan Branch (createOrphanBranch())

  • Checks if the branch already exists (idempotent)
  • Creates a temporary detached worktree with git worktree add --detach <tmpDir>
  • Inside that worktree, runs git checkout --orphan <branch> to create the orphan branch
  • Clears any tracked files with git rm -rf --ignore-unmatch .
  • Creates .gitkeep placeholder file
  • Commits with message "Initialize transcripts branch"
  • Removes the temporary worktree

Key Git Commands:

git worktree add --detach <tmpDir>        # detached mode for setup
git checkout --orphan transcripts         # create orphan branch
git rm -rf --ignore-unmatch .             # clear index
git commit -m "Initialize transcripts branch"
git worktree remove --force <tmpDir>      # cleanup

Step 2: Add Git Worktree (addWorktree())

  • Adds a persistent worktree at .transcripts/ pointing to the orphan branch
  • Command: git worktree add .transcripts/ transcripts
  • Skips if worktree already exists (idempotent)

Step 3: Update .gitignore (ensureGitignore())

  • Adds .transcripts/ entry to .gitignore (idempotent)

Step 4: Install Claude Code Hook (installClaudeHook())

  • Creates .claude/hooks/save-transcript.sh script (executable)
  • Updates .claude/settings.json to register SessionEnd hook
  • The hook is registered as async command type

Step 5: Install Git Post-Commit Hook (installPostCommitHook())

  • Writes or appends to .git/hooks/post-commit
  • Uses git rev-parse --git-common-dir to find the correct hooks directory (works in worktrees)
  • Uses markers # cg-transcripts-start and # cg-transcripts-end for idempotency

3. Git Integration Details

Orphan Branch Architecture

  • The "transcripts" branch is completely independent from the main branch history
  • It contains only the rendered transcripts, not affecting the main codebase history
  • Created using --orphan flag which makes it unrelated to any existing commits

Worktree Setup

  • .transcripts/ is a git worktree pointing to the orphan branch
  • The .transcripts/.git file is a worktree pointer (not a directory)
  • Worktree has its own git state independent of the main repository

Git-Dir Resolution

The post-commit hook uses git rev-parse --git-common-dir to find the shared git directory. This works correctly in both:

  • Regular repositories (returns .git)
  • Worktree contexts (returns the parent's .git)

Claude Code Hook (SessionEnd)

When a Claude Code session ends, the hook script runs:

Script Template (buildSaveTranscriptScript()):

#!/bin/bash
# Installed by cg install — renders Claude Code session transcripts to .transcripts/
set -e

INPUT=$(cat)  # receives JSON from Claude Code
TRANSCRIPT_PATH=$(echo "$INPUT" | jq -r '.transcript_path')
SESSION_ID=$(echo "$INPUT" | jq -r '.session_id')

# Safety checks: exit if no path or file missing
if [ -z "$TRANSCRIPT_PATH" ] || [ "$TRANSCRIPT_PATH" = "null" ]; then exit 0; fi
if [ ! -f "$TRANSCRIPT_PATH" ]; then exit 0; fi
if [ ! -d "$DEST_DIR" ]; then exit 0; fi

DEST_DIR="$CLAUDE_PROJECT_DIR/.transcripts"

# Step 1: Render the session to .transcripts/<session_id>/
cg render --agent claude --file "$TRANSCRIPT_PATH" --format html --format ... --out "$DEST_DIR/$SESSION_ID"

# Step 2: Update the manifest with metadata
cg manifest upsert --agent claude --file "$TRANSCRIPT_PATH" \
  --manifest "$CLAUDE_PROJECT_DIR/.transcripts/manifest.json" \
  --href "$SESSION_ID/index.html"

5. Post-Commit Hook (Git auto-commit)

Hook Script Location: .git/hooks/post-commit

Functionality:

# cg-transcripts-start
REPO_ROOT="$(git rev-parse --show-toplevel)"
WORKTREE="$REPO_ROOT/.transcripts"

if [ -d "$WORKTREE/.git" ] || [ -f "$WORKTREE/.git" ]; then
  # Step 1: Regenerate index from manifest
  cg index --dir "$WORKTREE" 2>/dev/null || true

  # Step 2: Get the main branch commit SHA for commit message
  MAIN_SHA="$(git rev-parse --short HEAD)"
  
  # Step 3: Unset git environment variables so we operate on worktree's repo
  unset GIT_DIR GIT_INDEX_FILE GIT_WORK_TREE
  
  # Step 4: Stage and commit any new transcripts in the worktree
  git -C "$WORKTREE" add -A 2>/dev/null
  git -C "$WORKTREE" diff --cached --quiet 2>/dev/null || \
    git -C "$WORKTREE" commit -m "transcripts @ $MAIN_SHA" --quiet 2>/dev/null || true
fi
# cg-transcripts-end

Key Features:

  • Runs after every git commit on the main repository
  • Automatically stages and commits any transcripts written to .transcripts/
  • Uses git -C to operate on the worktree without affecting the main repo
  • Generates a new index.html listing page from the manifest
  • Commit message includes the main repo's commit SHA for cross-referencing

8. Idempotency

All install steps are idempotent:

  • Branch creation checks if it exists before creating
  • Worktree only added if not present
  • Gitignore checks for existing entry before appending
  • Claude hook checks for duplicate entries before adding
  • Post-commit hook uses markers to prevent duplicate insertions
  • Multiple cg install runs are safe

This comprehensive system ensures that session transcripts are captured automatically, versioned separately from the main code, and automatically indexed for browsing!
