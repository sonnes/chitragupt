# Configuring Theme Variables in Tailwind CSS v4

## What is the @theme Directive?

The `@theme` directive is special CSS syntax that defines **design tokens** (called theme variables) which influence which utility classes exist in your project. Unlike regular CSS variables, theme variables do double duty:

1. They create CSS variables for use in your code
2. They instruct Tailwind to generate corresponding utility classes

## Basic Syntax

```css
@import "tailwindcss";

@theme {
  --color-mint-500: oklch(0.72 0.11 178);
}
```

This creates:
- A CSS variable: `var(--color-mint-500)`
- Utility classes: `bg-mint-500`, `text-mint-500`, `fill-mint-500`, etc.

## Why @theme Instead of :root?

Theme variables require explicit syntax because they do more than regular CSS variables. They must be:
- **Top-level only** (not nested under selectors/media queries)
- **Explicitly defined** to signal intent to Tailwind

Use `@theme` for design tokens that map to utilities, and `:root` for regular CSS variables that shouldn't have corresponding utility classes.

## Key Customization Patterns

### Extending the Default Theme
```css
@import "tailwindcss";

@theme {
  --font-script: Great Vibes, cursive;
}
```

### Overriding Default Values
```css
@theme {
  --breakpoint-sm: 30rem;  /* Changes sm: variant breakpoint */
}
```

### Replacing an Entire Namespace
```css
@theme {
  --color-*: initial;  /* Clear all default colors */
  --color-white: #fff;
  --color-purple: #3f3cbb;
  --color-midnight: #121063;
}
```

### Complete Custom Theme
```css
@theme {
  --*: initial;  /* Clear everything */
  --spacing: 4px;
  --font-body: Inter, sans-serif;
  --color-lagoon: oklch(0.72 0.11 221.19);
}
```

## Theme Variable Namespaces

| Namespace | Utilities |
|-----------|-----------|
| `--color-*` | Color utilities (`bg-red-500`, `text-sky-300`, etc.) |
| `--font-*` | Font family (`font-sans`, `font-serif`) |
| `--text-*` | Font size (`text-xl`) |
| `--breakpoint-*` | Responsive variants (`sm:*`, `md:*`) |
| `--spacing-*` | Padding, margin, sizing (`px-4`, `max-h-16`) |
| `--radius-*` | Border radius (`rounded-sm`) |
| `--shadow-*` | Box shadows (`shadow-md`) |
| `--animate-*` | Animations (`animate-spin`) |

## Advanced Features

### Referencing Other Variables (with `inline`)
```css
@theme inline {
  --font-sans: var(--font-inter);
}
```
The `inline` option prevents variable resolution issues by using the actual value.

### Defining Animation Keyframes
```css
@theme {
  --animate-fade-in-scale: fade-in-scale 0.3s ease-out;
  
  @keyframes fade-in-scale {
    0% {
      opacity: 0;
      transform: scale(0.95);
    }
    100% {
      opacity: 1;
      transform: scale(1);
    }
  }
}
```

### Using CSS Variables in Custom CSS
```css
@import "tailwindcss";

@layer components {
  .typography {
    p {
      font-size: var(--text-base);
      color: var(--color-gray-700);
    }
  }
}
```


# Dark Mode in Tailwind CSS v4

## Default Behavior

By default, Tailwind's dark mode uses the **`prefers-color-scheme` CSS media feature**, which respects the user's operating system color preference settings.

You can apply dark mode styles using the `dark:` variant:

```html
<div class="bg-white dark:bg-gray-800 rounded-lg px-6 py-8">
  <h3 class="text-gray-900 dark:text-white">Writes upside-down</h3>
  <p class="text-gray-500 dark:text-gray-400">
    The Zero Gravity Pen can be used to write in any orientation...
  </p>
</div>
```

## Custom Dark Mode Configuration

### Using a CSS Class

To control dark mode with a class instead of `prefers-color-scheme`, override the `dark` variant in your CSS:

```css
@import "tailwindcss";
@custom-variant dark (&:where(.dark, .dark *));
```

Then apply the `dark` class to your HTML:

```html
<html class="dark">
  <body>
    <div class="bg-white dark:bg-black"><!-- ... --></div>
  </body>
</html>
```

### Using a Data Attribute

Alternatively, use a data attribute:

```css
@import "tailwindcss";
@custom-variant dark (&:where([data-theme=dark], [data-theme=dark] *));
```

```html
<html data-theme="dark">
  <body>
    <div class="bg-white dark:bg-black"><!-- ... --></div>
  </body>
</html>
```

## Three-Way Theme Support

To support light mode, dark mode, AND system preference, use JavaScript with `localStorage` and `window.matchMedia()`:

```javascript
// Toggle based on user choice
document.documentElement.classList.toggle(
  "dark",
  localStorage.theme === "dark" ||
    (!("theme" in localStorage) && 
     window.matchMedia("(prefers-color-scheme: dark)").matches)
);

// User selects light mode
localStorage.theme = "light";

// User selects dark mode
localStorage.theme = "dark";

// User respects system preference
localStorage.removeItem("theme");
```

Important: @theme vs :root

In v4 CDN, use @theme (not :root) because:

  • @theme creates both CSS variables AND generates corresponding utilities
  • :root only creates variables, no utilities generated

<!-- INCORRECT: :root won't generate utilities -->
<style type="text/tailwindcss">
  :root {
    --color-primary: #2563eb;  /* Only CSS variable, NO utilities */

</style>
